## How to Contribute

I welcome contributions! Whether you want to add new chapters, improve existing notes, fix errors, or add diagrams, here's how to contribute:

### Getting Started

1. **Fork the repository** — Click the "Fork" button on GitHub to create your own copy.

2. **Clone your fork locally:**
   ```bash
   git clone https://github.com/YOUR_USERNAME/The-AVR-Microcontroller-and-Embedded-Systems-Using-Assembly-and-C-Notes.git
   cd The-AVR-Microcontroller-and-Embedded-Systems-Using-Assembly-and-C-Notes
   ```

3. **Create a feature branch:**
   ```bash
   git checkout -b feature/add-chapter-4
   # or
   git checkout -b fix/typo-in-chapter-3
   ```

### Making Changes

- **Add or update chapter notes** in markdown format (e.g., `chapter X.md`)
- **Place images** in the appropriate `assets/images/chapter-X/` folder
- **Keep formatting consistent** with existing chapters:
  - Use clear headings (H1, H2, H3)
  - Include code examples with proper syntax highlighting
  - Embed diagrams and reference images with descriptive alt text
  - Add table of contents for new chapters

### Submitting Changes

1. **Commit your changes** with clear, descriptive messages:
   ```bash
   git add .
   git commit -m "Add Chapter 4: Interrupts and Timers"
   ```

2. **Push to your fork:**
   ```bash
   git push origin feature/add-chapter-4
   ```

3. **Create a Pull Request** — Go to GitHub and open a PR against the `main` branch. Include:
   - A clear title describing your changes
   - A description of what was added, fixed, or improved
   - Any related issue numbers (if applicable)

### Types of Contributions Welcome

- ✅ New chapter notes from the book
- ✅ Diagrams, images, and visual references
- ✅ Clarifications or improvements to existing explanations
- ✅ Code examples and practical exercises
- ✅ Bug fixes and typo corrections
- ✅ Better organization or restructuring

### Before You Submit

- Ensure your markdown is properly formatted
- Verify all image links work correctly
- Check for spelling and grammatical errors
- Make sure images are clear and relevant to the content
- Test any code examples if applicable

### Questions?

Feel free to open an issue on GitHub to discuss ideas or ask questions before contributing.
